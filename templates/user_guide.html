1. Project Overview & Features

The Multi-School Smart ID System is a production-ready Flask web application designed for generating high-quality student ID cards in bulk or individually. It is built specifically for Schools, Colleges, Madrassas, Coaching Centers, Training Institutes, NGOs, and Education Trusts.

Core Capabilities

  • Smart Generation: Supports both portrait and landscape templates, multi-school tenanting, and printable HD output.
  • Multilingual Rendering: Flawless rendering of English, Urdu, Hindi, and Arabic, complete with RTL text support and dynamic font switching.
  • Dynamic QR & Barcode: Automatically generates Code128 barcodes and customizable QR codes with JSON, URL, or Text payloads.
  • Secure Administration: Includes template visual editing, student management, activity logging, and robust authentication.

2. AI Photo Processing & Export

The application leverages advanced machine learning models to fully automate the tedious process of student photo formatting.

  • Face-Aware Cropping: Utilizes Google's MediaPipe to automatically detect faces and perfectly center/crop photos, ensuring uniform ID cards.
  • Background Removal: Integrated with Rembg (ONNX Runtime) to cleanly strip away messy backgrounds.
  • Smart Trimming: Transparent edge trimming and hijab/turban-safe cropping rules ensure religious and cultural headwear isn't cut off.
  • Rounded Corners: Dynamically applies customizable corner radii and borders to the processed photo.

PDF Export Modes

When downloading batches for printing, the system offers two powerful PDF modes:

  • Editable PDF: Text and vector graphics (QR/Barcodes) remain fully editable. Highly compatible with CorelDRAW for print shops that need to make manual tweaks.
  • Print PDF (600 DPI): A high-quality raster rendering optimized for professional, direct-to-printer output without requiring CorelDRAW.

3. Tech Stack & Architecture

Built for scale and performance, the system architecture separates web serving from heavy image/PDF rendering.

Backend Framework

Python 3, Flask, SQLAlchemy, Gunicorn

Database

PostgreSQL (Production) / SQLite (Local)

Queue & Cache

Redis caching, RQ background processing

Media Processing

Pillow, OpenCV, PyMuPDF, ReportLab

Performance & Security

  • Optimizations: Redis caching with cache stampede protection, WEBP image optimization, lazy Redis connections, and ThreadPoolExecutor rendering.
  • Security Measures: CSRF Protection, secure password hashing, rate limiting, Redis-safe fallback modes, and comprehensive activity logging.
  • Storage: Integration with Cloudinary for scalable, cloud-based asset storage.

4. Admin Quick Start (10 minutes)

Step 1: Go to Settings tab and upload a card template (PDF/JPG/PNG).
Step 2: Select template language, text direction, and orientation.
Step 3: Open Template Settings, adjust fonts/positions/colors/photo area/QR, then click Save Template Settings.
Step 4: Generate one sample card to validate layout before bulk usage.
Step 5: Use Bulk Generate only after sample output is correct.
Tip: After changing settings, regenerate cards or previews. Existing generated images do not auto-update dynamically to save processing power.

6. Upload Template

Required inputs

  • School Name: identifies template group.
  • Template File: .pdf, .jpg, .jpeg, .png.
  • Language Script: English, Urdu, Hindi, Arabic.
  • Text Direction: LTR or RTL.
  • Card Orientation: Landscape or Portrait.
Recommendation: For Urdu/Arabic templates always use RTL and Unicode fonts to avoid broken glyphs and ensure correct cursor behavior.

8. Text Case, Language & Direction

Text case behavior

  • Text Case applies to both label text and value text during rendering.
  • Modes: normal, UPPERCASE, lowercase, Capitalize.
  • If you change this setting, regenerate the preview/card to see the effect.

Language + direction map

  • English / Hindi: Usually set to LTR (Left-to-Right).
  • Urdu / Arabic: Set to RTL (Right-to-Left).
Common mistake: Keeping direction LTR for Urdu/Arabic will severely break label/value alignment and character shaping.

9. Bulk Generation Workflow

  1. Prepare your Excel (XLSX) or CSV with required columns.
  2. Run a small test batch (2-5 rows) before a massive upload to verify mapping.
  3. Optionally upload a zip of photos; filenames should map cleanly to student identifiers.
  4. Watch the progress bar; do not close the tab during active generation as it interrupts background polling.
  5. Check sample outputs in the Preview tab before distributing final PDFs to printers.
Performance tip: The background task queue (RQ) handles heavy PDF processing so the web server doesn't crash. Large batches are perfectly safe once templates are finalized.

10. Premium Features

Template Versioning & Workflow

  • Every template save creates a snapshot. Rollback to any version from Premium → Workflow.
  • States: draft → review → approved → published.

Design QA Layer

  • Checks for overlaps, out-of-bounds text, hidden mandatory fields, missing photos, and low contrast.
  • Can enforce blocking rules before allowing bulk generation.

Advanced Batch Rules & Localization

  • Rules engine for empty-value handling (e.g., auto-hide empty Address rows).
  • Central localization dictionary JSON with per-side language lock rules.

Verification Portal 2.0

  • Issue signed, expiring verification links for parents/students to review cards digitally.
  • Revoke/unrevoke credential per student with full IP/Token audit trails.

Disaster Recovery & Analytics

  • Create/list snapshots, restore selected snapshot, or restore to a specific date/time point.
  • Analytics dashboard showing generation counts, revoked credentials, and AI photo fail rates.

11. Dynamic Form Fields

  • Each template can have custom fields dynamically generated (e.g., Blood Group, Roll Number, House, Session).
  • Field Name is the internal DB key (avoid spaces; use underscore_style).
  • Field Label is what appears on the generated card.
  • Display Order controls the vertical flow sequence on the card.
  • Use the select field type to provide strict drop-down options for users.

12. Data Management & Safety

Available Operations

  • Edit individual cards to fix typos without re-uploading spreadsheets.
  • Delete single student cards or wipe all cards for a specific template/school.
  • Run old file cleanup to free up Cloudinary storage space.
Safety order: Always perform an Export Backup before initiating bulk cleanup or delete operations.

13. Preview and Verification

  • Use the Preview tab to load a high-res 3D preview of any student card.
  • Use Regenerate Preview if you recently tweaked template settings.
  • Use the download action to fetch the single PDF for immediate printing.
  • Previews utilize cache-busting so you always see the freshest render after forcing a regeneration.

14. Advanced Styling & Corel Integration

Submission Deadline & Portal Locking

  • Set a strict date and time under Font Settings.
  • Once the deadline passes, students/parents are blocked from editing details or uploading new pictures from the public portal. Leave empty to allow continuous updates.

CorelDRAW Integration (Photo Export Modes)

  • Frame Only: Generates an empty vector rectangle in the PDF sheet. Use this when you intend to import the PDF into CorelDRAW and insert photos using the Corel PowerClip feature.
  • Embed Photo: Embeds the cropped student photo directly in the exported PDF sheet. Recommended for immediate printing without manual assembly in CorelDRAW.

Database Integrity Safeguards

  • Configure duplicate prevention rules from the Duplicate Check Settings.
  • Block student submissions if a duplicate Phone Number or duplicate Name + Class combination already exists, preventing double-printing of cards.

15. Troubleshooting

Issue Likely Cause Fix
Labels not changing with text case Old generated file still being cached Save settings and hit Regenerate on the preview/card output.
Urdu/Arabic alignment broken Wrong direction or unsupported font Set RTL direction and select a proper Unicode Arabic/Urdu font (e.g. Noto Nastaliq or Amiri).
Student blocked from editing records Deadline date has passed Check submission deadline field in admin settings; extend or clear it if needed.
Bulk generation errors Column mismatch or missing required fields Validate Excel column headers exactly match the system's expected names. Try a small test batch of 2 rows first.
Address lines overlapping Line height multiplier interpreted wrong Adjust Line Height setting in visual editor. Multipliers (e.g., 1.2) are now properly handled.
