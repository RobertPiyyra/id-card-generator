1. Quick Start (10 minutes)

Step 1: Go to Settings tab and upload a card template (PDF/JPG/PNG).
Step 2: Select template language, text direction, and orientation.
Step 3: Open Template Settings, adjust fonts/positions/colors/photo area/QR, then click Save Template Settings.
Step 4: Generate one sample card to validate layout before bulk usage.
Step 5: Use Bulk Generate only after sample output is correct.
Tip: After changing settings, regenerate cards or previews. Existing generated images do not auto-update.

4. Upload Template

Required inputs

  • School Name: identifies template group.
  • Template File: .pdf, .jpg, .jpeg, .png.
  • Language Script: English, Urdu, Hindi, Arabic.
  • Text Direction: LTR or RTL.
  • Card Orientation: Landscape or Portrait.
Recommendation: For Urdu/Arabic templates always use RTL and Unicode fonts to avoid broken glyphs.

6. Text Case, Language and Direction

Text case behavior

  • Text Case applies to both label text and value text during rendering.
  • Modes: normal, uppercase, lowercase, capitalize.
  • If you change this setting, regenerate preview/card to see effect.

Language + direction map

  • English: usually LTR
  • Hindi: usually LTR
  • Urdu: RTL
  • Arabic: RTL
Common mistake: Keeping direction LTR for Urdu/Arabic can break label/value alignment.

7. Bulk Generation Workflow

  1. Prepare CSV/XLSX with required columns first.
  2. Run a small test batch (2-5 rows) before full upload.
  3. Optionally upload photos; file names should map cleanly to student identifiers.
  4. Watch progress bar; do not close the tab during active generation.
  5. Check sample output in Preview tab before distributing final cards.
Performance tip: Large batches are safer after template coordinates are finalized.

8. Premium Features

Template Versioning + Rollback

  • Every template save/update creates a snapshot version.
  • You can rollback to a selected version from Premium → Workflow.

Approval Workflow

  • States: draft → review → approved → published.
  • State changes are logged in immutable audit events.

Design QA Layer

  • Checks overlap, out-of-bounds, hidden mandatory fields, missing photos, low contrast signals.
  • QA settings can enforce blocking before bulk generate and before compiled PDF export.

Advanced Batch Rules Engine

  • Supports rule configuration JSON for empty-value handling and layout/QR hints.
  • Rules are applied during bulk generation pipeline.

Parent/Student Verification Portal 2.0

  • Issue signed, expiring verification links.
  • Revoke/unrevoke credential per student.
  • Verification audit trail includes status/IP/token metadata.

Photo Intelligence Upgrade

  • On-demand quality score and pass/fail status per student photo.

Localization Pack Manager

  • Central localization dictionary JSON per template.
  • Per-side language lock rules enforce front/back language consistency.
  • Localized labels preview is available for front and back.

Import Mapping Studio

  • Save column mappings by template and validate against sample headers.
  • Bulk Ops supports selecting a mapping preset before generation.

Multi-tenant White Label (Template-level)

  • Branding config and print profile JSON storage in Premium → White Label.

Disaster Recovery Toolkit

  • Create/list snapshots, restore selected snapshot, or restore to a date/time point.

School Admin Analytics

  • Overview metrics for generation counts, revoked credentials, and photo quality fail counts.

9. Dynamic Form Fields

  • Each template can have custom fields like Blood Group, Roll Number, House, Session, etc.
  • Field Name is internal key (avoid spaces; use underscore style).
  • Field Label is what appears in form and on card.
  • Display Order controls vertical sequence on card.
  • Use select type with options for controlled values.
Best practice: Keep field names stable once bulk data is in use to avoid mapping issues.

10. Data Management and Safety

Available operations

  • Edit individual cards.
  • Delete single student cards.
  • Delete all cards for one template/school.
  • Run old file cleanup.
  • Delete all data and files (danger operation).
Safety order: Export backup first, then cleanup/delete operations.

11. Preview and Verification

  • Use Preview tab to load any student card from admin side.
  • Use regenerate preview when settings changed recently.
  • Use download action to fetch PDF for print/share.
  • If preview looks old, regenerate once and refresh browser.

Preview image URLs include cache-busting so newest image should load after regeneration.

12. Advanced Features & Custom Styling

Submission Deadline & Portal Locking

  • Set a strict date and time under Font Settings.
  • Once the deadline passes, students are blocked from editing details or uploading new pictures. Leave empty to allow continuous updates.

Advanced Photo Styling

  • Photo Border Color: Click the color picker in Photo Settings to set a custom frame color stroke around the student photo.
  • Corner Rounding: Set the corner radius in pixels. You can round all corners uniformly or specify unique values for the Top-Left, Top-Right, Bottom-Right, and Bottom-Left corners to create asymmetrical styles.

Corel Draw Integration (Photo Export Modes)

  • Frame Only: Generates an empty vector vector rectangle in the PDF sheet. Use this when you intend to import the PDF into CorelDraw and load photos using the Corel PowerClip feature.
  • Embed Photo: Embeds the cropped student photo directly in the exported PDF sheet. Recommended for immediate printing without manual assembly in CorelDraw.

Custom QR & Barcode Styling

  • QR Dot Styles: Toggle between Square Dots, Rounded Dots, Circular Dots, and Gapped Square to match the card theme.
  • Custom QR/Barcode Colors: Set custom color pickers for both the fill stroke and the background color. Ensure high color contrast (e.g. dark text/fill on white/light background) for reliable scanner reading.
  • Verification logo: Enable Include Logo in QR Center and set a path (e.g. logos/school_logo.png) to render a small school badge in the center of the QR code.

Text Separation and Case Modifications

  • Text Case: Set dropdown to UPPERCASE, lowercase, or Capitalize First Letters to sanitize user inputs automatically on the card.
  • Label Colon: Toggle Show Colon After Label off to hide colons. Toggle Align Colon Near Value on and adjust Colon Gap (px) to vertically align separating colons in a straight line.

Database Integrity Safeguards

  • Configure duplicate prevention rules from the Duplicate Check Settings card.
  • Choose to block student record submissions if a duplicate Phone Number or duplicate Name + Class combination already exists in the school database.

13. Troubleshooting

Issue Likely Cause Fix
Labels not changing with text case Old generated file still being viewed Save settings and regenerate preview/card output.
Urdu/Arabic alignment broken Wrong direction or unsupported font Set RTL and choose Unicode Arabic/Urdu font in template settings.
Preview card not scrolling / sticky sidebar broken Overflow restrictions in parent element Ensure template-settings wrapper has overflow: visible !important; set.
Student blocked from editing records Deadline date has passed Check submission deadline field in admin settings; extend or clear it if needed.
Bulk generation errors Column mismatch or missing required fields Validate columns and try a small test batch of 2-5 rows first.
