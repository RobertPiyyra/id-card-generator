1. Quick Start (10 minutes)

Step 1: Go to Settings tab and upload a card template (PDF/JPG/PNG).
Step 2: Select template language, text direction, and orientation.
Step 3: Open Template Settings, adjust fonts/positions/colors/photo area/QR, then click Save Template Settings.
Step 4: Generate one sample card to validate layout before bulk usage.
Step 5: Use Bulk Generate only after sample output is correct.
Tip: After changing settings, regenerate cards or previews. Existing generated images do not auto-update.

3. Upload Template

Required inputs

  • School Name: identifies template group.
  • Template File: .pdf, .jpg, .jpeg, .png.
  • Language Script: English, Urdu, Hindi, Arabic.
  • Text Direction: LTR or RTL.
  • Card Orientation: Landscape or Portrait.
Recommendation: For Urdu/Arabic templates always use RTL and Unicode fonts to avoid broken glyphs.

5. Text Case, Language and Direction

Text case behavior

  • Text Case applies to both label text and value text during rendering.
  • Modes: normal, uppercase, lowercase, capitalize.
  • If you change this setting, regenerate preview/card to see effect.

Language + direction map

  • English: usually LTR
  • Hindi: usually LTR
  • Urdu: RTL
  • Arabic: RTL
Common mistake: Keeping direction LTR for Urdu/Arabic can break label/value alignment.

6. Bulk Generation Workflow

  1. Prepare CSV/XLSX with required columns first.
  2. Run a small test batch (2-5 rows) before full upload.
  3. Optionally upload photos; file names should map cleanly to student identifiers.
  4. Watch progress bar; do not close the tab during active generation.
  5. Check sample output in Preview tab before distributing final cards.
Performance tip: Large batches are safer after template coordinates are finalized.

7. Dynamic Form Fields

  • Each template can have custom fields like Blood Group, Roll Number, House, Session, etc.
  • Field Name is internal key (avoid spaces; use underscore style).
  • Field Label is what appears in form and on card.
  • Display Order controls vertical sequence on card.
  • Use select type with options for controlled values.
Best practice: Keep field names stable once bulk data is in use to avoid mapping issues.

8. Data Management and Safety

Available operations

  • Edit individual cards.
  • Delete single student cards.
  • Delete all cards for one template/school.
  • Run old file cleanup.
  • Delete all data and files (danger operation).
Safety order: Export backup first, then cleanup/delete operations.

9. Preview and Verification

  • Use Preview tab to load any student card from admin side.
  • Use regenerate preview when settings changed recently.
  • Use download action to fetch PDF for print/share.
  • If preview looks old, regenerate once and refresh browser.

Preview image URLs include cache-busting so newest image should load after regeneration.

10. Troubleshooting

Issue Likely Cause Fix
Labels not changing with text case Old generated file still being viewed Save settings and regenerate preview/card output.
Urdu/Arabic alignment broken Wrong direction or unsupported font Set RTL and choose Unicode Arabic/Urdu font in template settings.
Manual button not opening Bad route mapping Open /static/user_guide.html directly.
Bulk generation errors Column mismatch or missing required fields Validate columns and try a small batch first.
